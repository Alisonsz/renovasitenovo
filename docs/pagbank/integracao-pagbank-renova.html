Requests e Responses — Integração PagBank

Renova Laser Depilação · Loja Laravel + Inertia/Vue · Integração via API de Pedidos (Orders) + Checkout PagBank

Base URL (produção): https://api.pagseguro.com

Base URL (sandbox, usada nos testes): https://sandbox.api.pagseguro.com

Autenticação: todas as chamadas usam o header Authorization: Bearer SEU_TOKEN e Content-Type: application/json.

Métodos de pagamento: PIX e Cartão de crédito (fluxo transparente via Orders API) e, opcionalmente, página de Checkout PagBank (que disponibiliza PIX, Cartão, Google Pay e Pagar com PagBank).

Observação de segurança: os exemplos abaixo usam dados fictícios — SEU_TOKEN, CPF de teste 111.444.777-35, cartão criptografado CARD_ENCRYPTED_TESTE, IDs ORDE_…/CHAR_…/CHEC_…. Nenhum dado real de cliente, token ou cartão está incluído.

Sumário

  1. Gerar chave pública (criptografia do cartão) — POST /public-keys
  2. Criar e pagar pedido com Cartão — POST /orders
  3. Criar pedido com PIX (QR Code) — POST /orders
  4. Consultar pedido (reconciliação de status) — GET /orders/{id}
  5. Checkout PagBank (página hospedada) — POST /checkouts
  6. Estorno/cancelamento de cobrança — POST /charges/{id}/cancel
  7. Webhook / Notificação de status (recebido)

1. Gerar chave pública (criptografia do cartão) Cartão

Antes de tokenizar o cartão no navegador (SDK PagBank), a aplicação gera a chave pública a partir do token.

Request

POST https://api.pagseguro.com/public-keys
Authorization: Bearer SEU_TOKEN
Content-Type: application/json

{
  "type": "card"
}

O PagBank envia um POST para a notification_url sempre que o status muda. Validamos a autenticidade pelo header x-authenticity-token.

Request recebido (PagBank → nosso servidor)

POST https://renovalaserdepilacao.com.br/webhooks/pagbank
x-authenticity-token: 8d09afba2eed749f2173cfd6faf16dc7f0228846ecf5ba1d7a11f33270d1513f
Content-Type: application/json

{
  "id": "ORDE_2B3C4D5E-0000-0000-0000-000000000002",
  "reference_id": "RL-20260607-000002",
  "created_at": "2026-06-07T10:05:00.000-03:00",
  "charges": [
    {
      "id": "CHAR_2B3C4D5E-0000-0000-0000-000000000002",
      "reference_id": "RL-20260607-000002",
      "status": "PAID",
      "paid_at": "2026-06-07T10:12:30.000-03:00",
      "amount": { "value": 50000, "currency": "BRL", "summary": { "total": 50000, "paid": 50000, "refunded": 0 } },
      "payment_method": { "type": "PIX" }
    }
  ]
}

Validação da autenticidade

hash_esperado = SHA-256( SEU_TOKEN + "-" + corpo_bruto_recebido )   // hex
valido = (hash_esperado === header "x-authenticity-token")

Response (nosso servidor → PagBank)

HTTP/1.1 200 OK        (ou 204 No Content)
